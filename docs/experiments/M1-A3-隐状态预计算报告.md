## 0. 与首版的关键区别（为什么重做）

首版 `precompute.py` 用 `np.linspace` 在序列里**均匀采样** n_seconds 个点当"决策位置"，
是**占位实现**，并非设计文档 §3.3 要求的"决策 token 处的 hidden state"。本次按判定标准
（计划 A3 Step 1）**精确实现**：决策 token 是单 token（`</silence>`=151669 / `</response>`=151670），
在整段 teacher-forcing 序列里按 id 定位，取 last-layer 该位置隐状态。

## 1. 方法（精确版，判定"通"的标准实现）

1. `build_second_by_second_messages(timeline)` 构造每秒 user/assistant 消息，
   assistant 带决策 token 前缀（回复 `</response>…`、沉默 `</silence>`）；
2. `apply_chat_template(..., add_generation_prompt=False)` → **整段 teacher forcing**，
   `model(**inputs, output_hidden_states=True)` **一次前向**（不逐秒 generate）；
3. `locate_decision_positions(input_ids)`：`np.isin(ids, (151669,151670))` 精确定位每秒决策 token；
4. 取 `outputs.hidden_states[-1][0]`（last-layer，[seq_len, 4096]）在决策位置的行 →
   `hidden_states.npy [N_decisions, 4096] float32`，并回写 `meta.d_model=4096`、`meta.n_decisions`。

## 3. Step 3 吞吐与全量估算（复核，数据大幅更新）

- **实测：0.19s/段（≈310 段/分·卡）**（模型加载 66.8s 一次性摊销后）。
  远快于 A2 的 generate，因为**只跑一次前向、不自回归解码**。
- **全量 10 万段估算**：
  - 单卡：100000 / 310 ≈ **5.4 卡时**；
  - **8 卡并行 ≈ 0.67 小时（约 40 分钟）**。

> 📉 首版估算"单卡 3 天 / 8 卡 1 天"是基于占位实现里错误的 2.5s/段；精确版实测 0.19s/段，
> 全量成本降到 **不足 1 卡·天**。隐状态获取这条 v1 语义主通道**不构成瓶颈**。
