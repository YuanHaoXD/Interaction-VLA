## 6. ⚠️ 供 §7.3「加量至 100 万」决策的关键数据

稀有簇的**绝对量**才是训练可用性的关键。按本次比例外推：

| 簇 | 10k 实测 | → 50 万条 | → 100 万条（§7.3） |
| --- | --- | --- | --- |
| attend | 625 | ~32,000 | ~64,000 |
| joy | 287 | ~14,700 | ~29,400 |
| think | 248 | ~12,700 | ~25,400 |
| surprise | 160 | ~8,200 | ~16,400 |
| affirm | 126 | ~6,500 | ~12,900 |
| fear | 44 | ~2,300 | ~4,500 |
| warm / sad | 29 / 27 | ~1,500 / ~1,400 | ~3,000 / ~2,800 |
| **annoy** | 11 | ~560 | ~1,100 |
| **negate / unsure / awkward** | 2 / 1 / 1 | ~100 / ~50 / ~50 | ~200 / ~100 / ~100 |

**判读**：
- 加量到 100 万，`joy`/`think`/`attend`/`surprise`/`fear` 都能进 ~1.6 万~6 万量级，**对语义区分度是实打实的补强**，$174 增量值得。
- 但 **`negate`/`unsure`/`awkward` 即使 100 万也只有 ~100~200 条**，加量救不了它们——这三簇在本语料里近乎不存在（视频播报语境本就极少否定/困惑/道歉）。它们只能靠 §7.4 的情绪语料或 §7.5 的训练侧加权兜底。**建议 Fable5 知悉：加量对这 3 簇无效。**
- §7.2 的免费再平衡（explain 事件下采样至 ≤70%）在本批数据上等价于：保留全部 1,561 条非 explain + 下采样 explain 至 ~3,642 条 → 事件集 ~5,203 条，非 explain 占比由 16% 提升到 **30%**，零成本翻倍。
