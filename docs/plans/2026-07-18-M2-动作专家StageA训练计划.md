| 阶段 | 条件 | 回答的问题 | 通过判据 |

## 契约 D-C1：训练样本格式（dataloader 输出，改动即大事）

```python
{
  "history":    float32 [25, 9],   # 过去 0.5s 轨迹（归一化后）
  "target":     float32 [75, 9],   # 未来 1.5s 动作块（归一化后，监督目标）
  "state_id":   int,               # 0=沉默 1=用户说话中 2=自己说话中（由 timeline 的 user_text/response 区间导出，区间=est_speech_dur_s）
  "idle_s":     float32,           # 已连续沉默秒数（无事件区间累计，供待机行为条件）
  "cluster_id": int,               # 0..12=13 簇，13=无事件（取窗口中心时刻所属事件的簇；密度控制未渲染的 response 仍算其簇）
  "phase":      float32,           # 事件内相位 0~1（无事件=0）
  "pending":    int,               # 1=有盯梢任务在身（user_text 后尚无 response），0=无
}
```

- 切窗：滑动步长 0.5s；每段掐头去尾 0.5s。归一化用数据集 `stats.json` 逐维 z-score，反归一化函数与之配对入库。
- 划分：按 **video_id 哈希** 切 train/val = 98/2（同视频不跨集，防泄漏）。
- 全部条件从 `timeline.json` + `meta.json` 推导，**零基座依赖**。

## Global Constraints

- 环境：服务器永久区仓库 + `PyTorch-2.6.0` conda env；新环境先 `bash smoke/setup_npu_env.sh`。
- 模型/优化：DiT 流匹配头（**抄 StarVLA** `flow_matching_head/`，`.cuda()`→`.to(device)`），总参数 30–60M；AdamW lr 1e-4、cosine、warmup 1k 步；bf16；EMA(0.999)；batch 256 chunk；梯度裁剪 1.0。
- **损失只有流匹配 MSE 一项**（x_t 线性插值、预测速度场）。不加平滑/加速度正则——扩散范式模仿分布，数据本身平滑；观测到问题再提"待决"。
- checkpoint/日志入永久区不入 git；每阶段结束报告入 `docs/experiments/`。

## 任务 D1：模型与训练循环（TDD）

**Files:** Create `expert/model.py`、`expert/train.py`、`tests/test_model.py`。
**Interfaces:** `ActionExpert(cond_dim=384)`；`forward(noised[75,9], t, cond) -> velocity[75,9]`；`encode_cond(batch) -> cond`。**cond 编码器必须留 `extra_cond: Optional[Tensor]` 入参**（Stage B 隐状态投影从这里进，缺省 None）。

- [ ] 测试：单 batch 前向/反传形状与数值健康；`encode_cond` 对 cluster_id=13（无事件）与 extra_cond=None 路径正确
- [ ] 训练循环：FM 损失、EMA、断点续训、tensorboard 标量（train/val loss）→ 1 个 batch 过拟合 100 步 loss 显著降 → commit

## 任务 D2：监控与采样套件

**Files:** Create `expert/sample.py`、`eval/quick_metrics.py`。

- [ ] `sample.py`：给定条件采样 chunk（Euler 10~20 步）；整段 rollout（滑动重叠 + temporal ensembling 加权拼接）→ 导出 episode 目录（actions.npy@50Hz，meta 标 `source:"generated"`），本机可直接 `sim/replay_episode.py` 回放
- [ ] `quick_metrics.py`：jerk 均值、limits_real 违例率、多样性（同条件采 8 次的成对距离均值）、FRMD-lite（用 val 集轨迹统计量的 Fréchet 距离，正式 FRMD 自编码器版留评测阶段）
- [ ] 挂进训练循环：每 2k 步跑一次写 tensorboard；每 10k 步导出 3 段 rollout 到交换目录供本机回放

## 任务 D3：A0 过拟合冒烟（单卡，半天出结论）

- [ ] 100 段子集、条件只留 history、训到 train loss 平台 → 采样回放（传回本机肉眼）
- [ ] **判定**：动作连续有呼吸感、无发散/抖动 → 进 D4；否则排查（归一化/时间步嵌入/学习率），卡 >3 天写待决

## 任务 D4：A1 运动先验（全量，8 卡）

- [ ] 验收：①jerk/违例达标；②多样性>0（同条件 8 采样不塌缩成一条）；③rollout 里可辨认：用户说话段有点头、长沉默有待机行为、pending 段不犯困；④报告含全部曲线与 3 段回放的本机观感记录

- [ ] 验收：固定其它条件只换 cluster_id 采样 → 13 簇动作族肉眼可辨（导出 13 段对比回放）；FRMD-lite 较 A1 降；A1/A2 消融表初版

## 降级预案

- FM 训不动（A0 失败超一周）→ 同 pipeline 换 MLP 回归头 + SmoothL1+速度/加速度正则（设计文档 v1 版），先拿底线结果，FM 转小规模排查。
- NPU 采样过慢 → 采样/评测迁 CPU（推理 50M 模型 CPU 可行），训练不动。
